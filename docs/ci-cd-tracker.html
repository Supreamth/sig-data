Rollback Strategy

Fast UI Rollback

Restore index-v1.html as root without any container rebuild or deploy. Fastest option — seconds.

cp echarts-dashboard/public/index-v1.html \ echarts-dashboard/public/index.html docker compose --profile stack up -d --build echarts-dashboard

Git Rollback

Revert the Task 13 root-switch commit. Keeps V2 files in repo for re-attempt.

git revert <task-13-commit-sha> git push origin feature/sigen-energy-cockpit-v2

Container Rollback

Redeploy previous tag via existing production release workflow. Rollback targets: v1.0.1 → v1.0.0.

# Via GitHub Actions deploy-production.yml # Set tag=v1.0.1 and dry_run=false

Pre-Switch Safety

V1 root is never overwritten until Task 13. At any point before that, index.html = V1 and rollback is a no-op.
